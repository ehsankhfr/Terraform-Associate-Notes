## ðŸ§ª Type Constraints in Terraform

Terraform uses **types** to describe variables and outputs. This helps catch errors early and makes code more understandable.

---

### ðŸ”¹ Primitive Types

| Type   | Description |
|--------|-------------|
| `string` | A plain text string. Example: `"hello"` |
| `number` | A numeric value. Example: `42`, `3.14` |
| `bool` | A boolean (`true` or `false`). |

---

### ðŸ”¸ Complex Types

#### ðŸ”¸ Collection Types

Used to store **multiple values of the same type**.

| Type | Description | Example |
|------|-------------|---------|
| `list(type)` | An ordered sequence. | `list(string)` â†’ `["a", "b", "c"]` |
| `map(type)` | A collection of key-value pairs. | `map(string)` â†’ `{ name = "John", age = "30" }` |
| `set(type)` | An unordered collection with unique values. | `set(number)` â†’ `{1, 2, 3}` |

> âœ… `list` keeps order. `set` doesn't. Use `map` when you need key-value lookup.

---

#### ðŸ”¸ Structural Types

Used when you want to store **values of different types**.

| Type | Description | Example |
|------|-------------|---------|
| `object({key=type, ...})` | A group of named attributes with types. | `object({name=string, age=number})` |
| `tuple([type1, type2, ...])` | A fixed-length list of values with specific types. | `tuple([string, number, bool])` |

```hcl
variable "person" {
  type = object({
    name = string
    age  = number
  })
}
```

---

### ðŸŒ€ Special Type: `any`

| Type | Description | Example |
|------|-------------|---------|
| `any` | Accepts any type; resolved at runtime. | `variable "env" { type = any }` |

> âš  Use `any` with caution. It removes static type safety.

## âœ… Real-World Examples

### Example: Using Functions
```hcl
variable "env_name" {
  default = "PROD"
}

output "lower_env" {
  value = lower(var.env_name)
}
```
**Output:** `"prod"`

---

### Example: Using JSON
```hcl
variable "json_input" {
  default = "{\"enabled\": true}"
}

output "enabled_status" {
  value = jsondecode(var.json_input)["enabled"]
}
```
**Output:** `true`

---

### Example: Using Object Type
```hcl
variable "settings" {
  type = object({
    port    = number
    enabled = bool
  })
  default = {
    port    = 8080
    enabled = true
  }
}

output "port" {
  value = var.settings.port
}
```

---

### Example: Using Tuple
```hcl
variable "tuple_var" {
  type = tuple([string, number, bool])
  default = ["app", 3, false]
}

output "tuple_example" {
  value = var.tuple_var[1]
}
```
**Output:** `3`

---

### Example: Merge Tags and Use Defaults
```hcl
variable "default_tags" {
  default = {
    team = "devops"
    env  = "staging"
  }
}

variable "extra_tags" {
  default = {
    project = "terraform"
    env     = "prod"
  }
}

output "merged_tags" {
  value = merge(var.default_tags, var.extra_tags)
}
```
**Output:** `{ team = "devops", env = "prod", project = "terraform" }`
